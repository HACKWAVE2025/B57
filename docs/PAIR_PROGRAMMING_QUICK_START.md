## ğŸ¯ Common Workflows

### Bug Fix Session

1. Create session: "Bug Fix - User Login"
2. Share context in chat
3. Navigate to bug location
4. Driver implements fix
5. Navigator reviews code
6. Save snapshot when complete
7. Download fixed code

### Learning Session

1. Create session: "Learning React Hooks"
2. Driver writes code slowly
3. Navigator asks questions
4. Switch roles frequently
5. Save snapshots of examples
6. Review history later

### Code Review

1. Create session with existing code
2. Navigator reviews line by line
3. Discuss improvements in chat
4. Driver makes changes
5. Save final version

## ğŸ“ Learning Resources

### Pair Programming Styles

1. **Ping Pong**
   - One writes test
   - Other writes code
   - Switch roles

2. **Driver-Navigator**
   - Driver codes
   - Navigator reviews
   - Regular switches

3. **Strong Style**
   - Navigator directs
   - Driver implements
   - Forces communication

### Recommended Timing

- **Pomodoro Style**: Switch every 25 min
- **Feature-based**: Switch per feature
- **Time-boxed**: Switch every 15-30 min
- **Natural breaks**: Switch at logical points
